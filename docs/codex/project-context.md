# Project Context

## What This Repository Is
This repository contains a beginner-friendly personal portfolio site served through Express.

The portfolio is intentionally positioned as:
- QA-first
- recruiter-friendly
- simple to maintain
- expandable later with stronger proof such as resume links, certifications, and QA case studies

## Technical Shape
- Server entry: `server.js`
- Test command: `npm test`
- Main page: `public/index.html`
- Main styles: `public/css/main.css`, `public/css/components.css`, `public/css/pages.css`
- Frontend behavior: `public/js/main.js`, `public/js/navigation.js`, `public/js/sections.js`
- Static assets: `public/assets/images/`
- Detail pages: `public/projects/`

## Current Content Direction
- homepage explains QA-first identity
- development skills support the QA profile
- project pages summarize real academic/development work
- some areas are placeholders until real content is supplied by the repo owner

## Important Source Material
- `Github-Readme.md` is the owner's GitHub profile base and can be used as reference material
- it should not automatically override the wording or structure of the portfolio

## What To Preserve
- simple file organization
- honest professional claims
- portable relative paths in HTML
- minimal Express setup
